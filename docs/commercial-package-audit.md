# Commercial Package Audit

This is the short audit of the current commercial documentation package for ORI
CRUIT HUB.

The purpose is to answer four questions:

1. what is already coherent
2. what is duplicated
3. what should be treated as active
4. what should be treated as secondary reference

## 1. What is already coherent

The current package is commercially coherent on the most important points:

- the motion is consistently framed as guided demo -> guided pilot -> paid rollout
- broad self-serve SaaS positioning is consistently avoided
- pilot pricing is consistent across the main active assets
- rollout entry pricing is consistent across the main active assets
- the pipeline language is now explicit and operational
- proposal and close control is now explicit
- pilot handoff logic exists

This means the package is already usable for real outbound and first pilot conversion.

## 2. Consistency confirmed

These points are repeated consistently across the current active package:

- guided pilot setup: `EUR 2,500`
- guided pilot cycle: `EUR 2,000 / 30 days`
- rollout entry anchor: `EUR 1,250 / month`
- pricing discipline: reduce scope before discount
- positioning rule: do not present the offer as mature self-serve SaaS

This is important because pricing inconsistency would immediately weaken commercial control.

## 6. What could be consolidated later

Not urgent, but sensible later:

1. archive or relabel older draft assets
2. mark clearly which docs are “active”, “support”, and “archive”
3. remove ambiguity between old and final one-pager versions
4. consider one single pricing hub that links all pricing sub-docs

This is cleanup work, not a blocker.

## 7. What should not be touched yet

Do not over-edit the package right now.

The commercial system has just reached a usable, coherent state.

At this stage, preserving momentum matters more than over-optimizing document structure.

## 8. Final conclusion

Current status:

- commercially usable
- pricing-consistent
- operationally structured
- ready for disciplined outreach and first pilot closing

The next improvement should be execution, not more theory.
