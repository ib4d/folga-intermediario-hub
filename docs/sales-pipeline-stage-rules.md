# Sales Pipeline Stage Rules

This document defines the only pipeline stages that should be used for the
current guided commercial motion of ORI CRUIT HUB.

The goal is simple:

- keep pipeline language consistent
- prevent fake progress
- make next actions explicit
- protect time from weak opportunities

## 1. Stage list

Use only these stages:

1. target identified
2. first contact prepared
3. first contact sent
4. replied
5. demo booked
6. demo completed
7. pilot qualified
8. proposal sent
9. decision pending
10. won
11. nurture
12. disqualified / lost

Do not invent extra stages unless the commercial motion changes materially.

## 2. General pipeline rules

Every account in the pipeline must always have:

- one owner
- one current stage
- one last action date
- one next action date
- one explicit next action

If one of these is missing, the account is not properly managed.

## 3. Stage definitions

### 1. target identified

Meaning:

- the account fits the commercial profile
- it belongs in the active target list

Entry rule:

- the account has been selected as a real target

Exit rule:

- a first contact path has been prepared

Do not stay here if the account is already being researched for contact.

### 2. first contact prepared

Meaning:

- contact route is known
- message angle is chosen
- the first touch is ready to send

Entry rule:

- contact or route exists
- there is enough context to personalize outreach

Exit rule:

- the first touch has actually been sent

Do not move here if there is still no credible contact path.

### 3. first contact sent

Meaning:

- first outreach has been sent
- follow-up timing is already defined

Entry rule:

- outreach was actually sent
- the send has been logged

Exit rule:

- the account replies
- or a follow-up / no-response outcome forces a different stage

Never leave an account here without a next follow-up date.

### 4. replied

Meaning:

- the account responded in a way that requires evaluation

Entry rule:

- there is a real reply, not just an auto-response

Exit rule:

- demo booked
- nurture
- or disqualified / lost

This stage should usually be short.

### 5. demo booked

Meaning:

- a real meeting is scheduled

Entry rule:

- date exists
- attendees are known
- purpose of the meeting is clear

Exit rule:

- demo completed
- or back to replied/nurture if the meeting collapses

Do not count “let’s speak sometime” as demo booked.

### 6. demo completed

Meaning:

- the conversation happened
- qualification evidence has been gathered

Entry rule:

- the demo was actually held
- notes were captured

Exit rule:

- pilot qualified
- nurture
- or disqualified / lost

A completed demo without clear notes is operationally incomplete.

### 7. pilot qualified

Meaning:

- the account has a real pain
- a concrete workflow is in scope
- there is a plausible path to a paid guided pilot

Entry rule:

- pain is explicit
- user/team is identifiable
- target start window is plausible
- buyer path is visible enough to continue

Exit rule:

- proposal sent
- nurture
- or disqualified / lost

Do not mark pilot qualified based on enthusiasm alone.

### 8. proposal sent

Meaning:

- a commercial proposal has been delivered

Entry rule:

- scope is defined
- pilot framing is clear
- pricing was intentionally sent

Exit rule:

- decision pending
- won
- or disqualified / lost

If pricing was sent without scope, that is not a healthy proposal stage.

### 9. decision pending

Meaning:

- the account has the proposal and an explicit decision path exists

Entry rule:

- follow-up date is known
- buyer-side next step is known

Exit rule:

- won
- nurture
- or disqualified / lost

Do not leave accounts here without a real decision checkpoint.

### 10. won

Meaning:

- the client has agreed to proceed with the paid pilot or rollout

Entry rule:

- commercial yes is explicit
- next onboarding/commercial step is defined

Exit rule:

- none in sales pipeline

At this point the account should transition into onboarding execution.

### 11. nurture

Meaning:

- the account is not dead, but it is not active enough for current pipeline effort

Entry rule:

- timing is weak
- pain is not urgent enough
- or authority/readiness is incomplete

Exit rule:

- reactivated into a normal active stage
- or disqualified / lost

Nurture is not a parking lot for forgotten accounts.

### 12. disqualified / lost

Meaning:

- the opportunity should not consume active effort now

Entry rule:

- poor fit
- no pain
- no owner
- no timing
- no budget path
- or clear rejection

Exit rule:

- only if materially new information appears later

Use this stage aggressively enough to keep pipeline honest.

## 4. Exit discipline

An account should move stage only if the exit rule of the current stage is true.

Bad examples:

- moving to demo completed without notes
- moving to pilot qualified because the call felt positive
- moving to proposal sent after only discussing pricing loosely
- leaving a stale account in decision pending for weeks

## 5. Maximum tolerated stagnation

Use these limits as operating discipline:

- first contact prepared: 3 working days max
- first contact sent: 5 working days without follow-up update
- replied: 3 working days max without clear next step
- demo completed: same-day note capture, 2 working days max to stage decision
- proposal sent: 5 working days max without follow-up action
- decision pending: 10 working days max without explicit buyer movement

If a stage exceeds these limits, review the account immediately.

## 6. What counts as a healthy pipeline

A healthy pipeline is not a big pipeline.

A healthy pipeline is one where:

- stages mean the same thing every time
- next actions are real
- weak accounts are removed fast
- strong accounts move forward with urgency

## 7. Weekly review questions

Review the pipeline weekly with these questions:

1. Which accounts moved forward by one real stage?
2. Which accounts are stale and should be nurtured or disqualified?
3. Which demos actually created pilot-qualified opportunities?
4. Which proposals have real decision dates?
5. Where are we pretending progress instead of creating it?

## 8. Rule to remember

Pipeline should reflect reality, not optimism.
