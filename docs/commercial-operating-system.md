# Commercial Operating System

This is the unified operating manual for the current guided commercial motion of
ORI CRUIT HUB.

Use it as the single reference for:

- target selection
- outreach execution
- CRM discipline
- demo qualification
- proposal control
- close management
- pilot handoff

The operating principle is simple:

- fewer accounts
- better fit
- cleaner movement
- stronger pricing control

## 1. Commercial motion

The current motion is:

1. identify high-fit target accounts
2. run focused outreach
3. book and run guided demos
4. qualify for a paid pilot
5. send controlled proposal
6. convert into paid pilot
7. hand off into guided onboarding

Do not position the product as broad self-serve SaaS.

## 3. Target account rule

Only work accounts that fit at least most of these conditions:

- international recruitment or foreign-worker operations
- document-heavy process
- coordination across recruitment, legal, logistics, or branch teams
- enough volume or complexity to justify workflow tooling
- realistic B2B buyer path

Avoid filling the pipeline with weak-fit small accounts just to create activity.

## 4. Outreach rule

Outreach must be:

- account-specific
- pain-led
- short
- operational
- logged immediately

Do not lead with:

- full product explanation
- pricing
- generic automation language
- excessive feature lists

The first objective is not to sell the whole platform.

The first objective is to earn a real conversation.

## 5. CRM rule

Every active account must always show:

- owner
- current stage
- last action date
- next action date
- next action
- short notes

No account should be touched without immediate CRM update.

## 6. Pipeline rule

The approved stage model is defined in:

- `docs/sales-pipeline-stage-rules.md`

Use only that stage language.

The most important control rule is:

- pipeline must reflect reality, not optimism

That means:

- stale accounts are reviewed fast
- weak opportunities are disqualified fast
- demo enthusiasm is not qualification
- proposal is not progress unless decision path exists

## 7. Demo rule

The demo exists to confirm whether a paid pilot is justified.

A good demo must validate:

- real workflow pain
- real user/team
- real operational relevance
- real buyer path
- plausible timing

If these are not visible, the result should be nurture or disqualify, not forced advancement.

Use:

- `docs/discovery-demo-notes-template.md`
- `docs/demo-to-pilot-conversion-checklist.md`

## 8. Proposal rule

Do not send proposals early.

A proposal is allowed only when:

- the opportunity is already pilot-qualified
- scope is concrete
- pain is explicit
- pilot framing is understood
- the buying path is visible enough to ask for decision

Use:

- `docs/short-commercial-proposal.md`
- `docs/proposal-and-close-control.md`

## 9. Pricing rule

Pricing discipline is part of the sales system.

Current rule:

- anchor high
- defend value
- reduce scope before discount
- protect the floor

Use:

- `docs/pilot-pricing-sheet.md`
- `docs/paid-rollout-pricing-sheet.md`
- `docs/pricing-policy-guided-launch.md`
- `docs/pricing-objection-handling.md`

Never discount as a reflex.

## 10. Close rule

Every qualified opportunity must end in one of these states:

1. won
2. nurture
3. disqualified / lost

There should be no indefinite middle ground.

If an account sits in decision pending without real buyer movement, it is a control failure.

## 11. Pilot handoff rule

Once the account is won:

- sales should stop improvising
- onboarding should start with explicit handoff

Use:

- `docs/pilot-onboarding-handoff-template.md`
- `docs/guided-pilot-operator-runbook.md`

The handoff must define:

- client owner
- internal owner
- scope
- dates
- success criteria
- agreed commercial structure

## 12. Weekly operating review

Run one weekly review against the whole commercial system.

Answer:

1. Which accounts moved one real stage this week?
2. Which accounts are stale?
3. Which accounts should be nurtured?
4. Which accounts should be disqualified?
5. Which demos created true pilot-qualified opportunities?
6. Which proposals have a real decision date?
7. Which open opportunities are strongest by evidence, not feeling?

If this review does not happen, pipeline quality will decay quickly.

## 13. Metrics that matter now

For the current stage, track only a few metrics:

- first contacts sent
- reply rate
- demos booked
- demos completed
- pilot-qualified opportunities
- proposals sent
- wins

Second-level metrics:

- reply-to-demo rate
- demo-to-pilot-qualified rate
- proposal-to-win rate

Do not overbuild reporting before real commercial repetition exists.

## 14. Failure modes to watch

The commercial system is breaking when:

- outreach is being sent without logging
- stages mean different things to different people
- demos are run without qualification notes
- pricing is sent before scope is real
- weak accounts stay active too long
- proposals go out without decision control
- won accounts move into onboarding without clean handoff

These failures create fake pipeline and wasted effort.

## 15. Minimal stack for execution

The system is already sufficient if the team uses:

- the CRM sheet
- the outreach packs
- the demo qualification documents
- the pricing discipline documents
- the proposal control document
- the pilot handoff template

Nothing more is required to start commercial repetition.

## 16. Rule to remember

This commercial system should create:

- fewer but better conversations
- fewer but stronger proposals
- fewer but more serious pilots

That is the right shape for the current stage.
