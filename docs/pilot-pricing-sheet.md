# Pilot Pricing Sheet

This is the minimum pricing structure for the current commercial stage.

It is designed for guided pilots, not for self-serve SaaS.

## 1. Pricing model purpose

The pilot price should do three things:

- create commercial seriousness
- keep onboarding effort justified
- avoid pretending the product is already a fully automated SaaS motion

## 2. Recommended pilot price structure

Use a simple structure with only two parts:

1. setup / guided onboarding fee
2. pilot period fee

Example commercial logic:

- setup fee: covers guided onboarding, sandbox preparation, first operator alignment
- pilot fee: covers the live pilot cycle and support cadence

## 3. What the pilot should include

The pilot offer should explicitly include:

- one active tenant
- one customer-side owner
- one guided onboarding cycle
- one agreed workflow scope
- one review cadence during the pilot
- limited rollout support for the first real operating use

## 4. What the pilot should not imply

Do not imply:

- unlimited support
- enterprise custom development
- fully productized self-serve billing
- production-wide customization without commercial review

## 5. Recommended pricing logic

Use pricing based on:

- onboarding effort
- number of operators
- expected pilot scope
- operational complexity of the customer

Do not overcomplicate the first offers.

## 6. Recommended internal pricing fields

Recommended list price:

- pilot setup fee: `€2,500`
- pilot cycle fee: `€2,000 / 30 days`
- pilot duration default: `30 days`, extendable only by explicit decision
- included operator seats: `up to 5`
- support cadence included: `1 onboarding session + 1 weekly review call`

Recommended early-adopter floor:

- setup fee floor: `€1,500`
- pilot cycle floor: `€1,250 / 30 days`

Rule:

Do not open below the floor unless there is a very strong logo, reference, or
distribution reason.

## 7. Positioning line

Use:

`The pilot is a guided commercial phase designed to validate the workflow with your real team before a broader rollout.`

## 8. Current decision

Recommended motion:

`Charge pilots in a simple guided format. Do not wait for self-serve billing to start commercial validation.`
